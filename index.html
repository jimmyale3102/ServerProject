Active Directory

Active Directory (AD) es un servicio de directorio para su uso en un entorno Windows Server. Se trata de una estructura de base de datos distribuida y jerárquica que comparte información de infraestructura para localizar, proteger, administrar y organizar los recursos del equipo y de la red, como archivos, usuarios, grupos, periféricos y dispositivos de red.


Active Directory es el servicio de directorio propietario de Microsoft para su uso en redes de dominio de Windows. Cuenta con funciones de autenticación y autorización y proporciona un framework para otros servicios similares. Básicamente, el directorio consiste en una base de datos LDAP que contiene objetos en red. Active Directory utiliza el sistema operativo Windows Server.

Instalación y Configuración

1. Antes de comenzar a instalar nuestros servicios, debemos establecer una IP estática.

para eso, nos dirigimos a las propiedades de la red y en este caso será 192.168.13.1. Por lo tanto, la máscara de red es 255.255.255.0.

2. Iniciamos la instalación de los roles y características que se van a usar para hacer uso de Active Direectory.

3. Al finalizar la instalación, se procede al link que genera llamado: Promover este servidor a controlador de dominio.

4. En este momento se debe agregar un nuevo bosque, en el cual se debe especificar el nombre de domimnio raíz. con este nombre de dominio vamos a trabajar de ahora en adelante.

5. Por último, se debe agregar una contraseña de modo de restauración de servicios de directorio (DSRM).

Domain Name System

El sistema de nombres de dominio (Domain Name System o DNS, por sus siglas en inglés) es un sistema de nomenclatura jerárquico descentralizado para dispositivos conectados a redes IP como Internet o una red privada. Este sistema asocia información variada con nombre de dominio asignado a cada uno de los participantes. Su función más importante es "traducir" nombres inteligibles para las personas en identificadores binarios asociados con los equipos conectados a la red, esto con el propósito de poder localizar y direccionar estos equipos mundialmente.​

Instalación y Configuración

1. Como en la sección de Active Directory se creó el dominio con el que vamos a trabajar, en esta parte vamos a configurar el servidor DNS. Se procede a la sección de Administración de DNS. Aquí, vamos a agregar una nueva zona

2. Luego, vamos a ingresar el nomobre de la zona, que en este caso es www.alejo.dev. Con este nombre, va a actuar el servidor de autorización.

3. Ahora, nos dirigimos a nuestro nombre de dominio creado anteriormente con la zona, y vamos a agregar un host nuevo. Esto se puede encontrar en las opciones desplegadas al hacer click derecho. ahí lo único que se va a agregar es la dirección IP

IIS (Internet Information Services)

Son las iniciales de Internet Information Services y si bien es más conocido como servidor web en realidad son un conjuntos de servicios que transforman un sistema Microsoft Windows en un servidor capaz de ofrecer servicios Web, FTP y SMTP entre otros.

En el caso de SMTP no está pensado como un servidor de correo completo sino simplemente como herramienta a disposición de las aplicaciones web alojadas para que puedan enviar correos electrónicos diferentes destinatarios pero no recibir correo de otros proveedores ya que no incorpora los protocolos IMAP/ṔOP.

Instalación y Configuración

1. Como en pasos anteriores, se instalan los roles y características, en este caso el servidor web (IIS).

2. En esta sección de seleccionar roles, activamos el servidor FTP con sus respectivos roles asociados.

3. Al finalizar la instalación. En el panel izquierdo nos dirigimos al servidor IIS y entramos al administrador de Internet Information Services..

4. En la ruta C/inetpub/wwwroot agregamos nuestro proyecto web.

5. Ahora en el administrador IIS, en la sección de Sitios dentro del servidor principal, agregamos un sitio web. Se asigna un nombre, se selecciona la ruta de acceso al código fuente (Ruta donde está ubicado el proyecto), Se selecciona el tipo http, se agrega la dirección ip del servidor, y finalmente la dirección del host.

Dynamic Host Configuration Protocol

El DHCP se desarrolló como solución para redes de gran envergadura y ordenadores portátiles y por ello complementa a BOOTP, entre otras cosas, por su capacidad para asignar automáticamente direcciones de red reutilizables y por la existencia de posibilidades de configuración adicionales.

La asignación de direcciones con DHCP se basa en un modelo cliente-servidor: el terminal que quiere conectarse solicita la configuración IP a un servidor DHCP que, por su parte, recurre a una base de datos que contiene los parámetros de red asignables.​

Instalación y Configuración

1. Iniciamos instalando DHCP, de la misma forma que hemos instalado las anteriores caracteristicas, de la misma manera leer los pasos detalladamente y terminar la instalación.

2. Una vez finalizada la instalación, nos dirigimos a la sección para completar la configuración de DHCP

3. Aquí, se deben asignar las credenciales que se usarán como autorización entre DHCP y AD DS. Se dejan las que aparecen or defecto.

4. Finalmente nos indican los estados de los pasos que se hicieron anteriormente. aquí solo daremos en cerrar..

5. Vamos a la sección de herramientas en el navbar, y nos dirigimos a la sección de DHCP.

6. Una vez aquí, nos dirigimos en el menú lateral izquierdo a la sección de IPv4 dentro de nuestro servidor. Aquí agregamos un nuevo ámbito.

7. En el asistente de creación de un nuevo ámbito debemos ingresar el nombre del mismo. En este caso es Group.

8. Luego, se debe ingresar un rango de direcciones IP, esto nos va a indicar la cantidad de clientes que se podrán conectar a nuestro servidor.

9. Ahora se debe agregar la dirección de la puerta de enlace..

10. Ahora se debe agregar el nombre del servidor, y luego dar en la opción de "Resolver", esto para que se obtenga la dirección Ip del servidor (la misma que asignamos como estática).

11. Finalmente, esa dirección IP que se generó en el paso anterior, la agregamos a la lista que se tiene y damos en siguiente.

File Transfer Protocol

El protocolo FTP (File Transfer Protocol), o Protocolo de transferencia de archivos por su traducción al español, es un protocolo de red utilizado para la transferencia de archivos entre las computadoras conectadas a una red TCP. En la actualidad son muchos los usuarios que han reemplazado al FTP por servicios de almacenamiento en línea como Dropbox, Google Drive u OneDrive, sobre todo por la comodidad de uso y transparencia que estos últimos ofrecen.

Instalación y Configuración

1. Como ya se instalaron los roles y características en pasos anteriores del IIS, entonces creamos una carpeta en la ruta C/inetpub/ftproot/ llamada dtpTest, la cual contiene un archivo de texto llamado test.ftp.

2. Ahora en la ventana de Administrador de Internet Information Services, creamos un sitio FTP a nivel del servidor.

3. como nombre del sitio dejamos ftptest, y como ruta elegimos la misma donde creamos la carpeta ftpTest.

4. Luego, se debe ingresar la dirección IP, en este caso la del servidor. También elegimos la opción de iniciar sitio FTP automáticamente sin SSL.

5. Se elige la autenticación anónima, se permite el acceso a todos los usuarios y finalmente se dan permisos de lectura y escritura.

6. Como resultado, abrimos el navegador Internet Explorer, y digitamos en la barra de búsqueda ftp://192.168.13.1 y se puede visualizar los archivos que se compartieron.

Simple Mail Transfer Protocol

El protocolo para transferencia simple de correo es un protocolo de red utilizado para el intercambio de mensajes de correo electrónico entre computadoras u otros dispositivos (PDA, teléfonos móviles, impresoras, etc).Son estándares oficiales de Internet que fueron reemplazados respectivamente por el RFC 2821 y el RFC 2822, que a su vez lo fueron por el RFC 5321 y el RFC 5322.1​

El funcionamiento de este protocolo se da en línea, de manera que opera en los servicios de correo electrónico. Sin embargo, este protocolo posee algunas limitaciones en cuanto a la recepción de mensajes en el servidor de destino (cola de mensajes recibidos). Como alternativa a esta limitación se asocia normalmente a este protocolo con otros, como el POP o IMAP, otorgando a SMTP la tarea específica de enviar correo, y recibirlos empleando los otros protocolos antes mencionados (POP O IMAP)..

Instalación y Configuración

1. Para el uso de SMTP, se instaló Mail Enabled. Luego de esto, se procede a abrir la aplicación. En la sección de Servers -> localhost -> Services and connectors -> SMTP entramos a las propiedades y agregamos un nombre de dominio local. y la dirección del DNS.

2. En la parte de default mail domain name, se agrega la dirección "mail.alejo.dev

3. Finalmente, se aplican los cambios y se cierra la ventana.

4. Ahora, en la sección de Servers -> localhost -> System -> Services status podemos ver todos los servicios, y en el menú lateral derecho, elegimos Restart all services.

5. Ahora en la ruta Messaging Manager -> Post offices -> alejo.dev -> Mailboxes Creamos un nuevo Mailbox, que en este caso son las cuentas de correo electrónico de los usuarios. Básicamente se crea con un nombre y una contraseña.
Ese nombre al final es alejandro@alejo.dev como correo electrónico

6. Por último, en la ventana de configuración del IIS, entramos al MailEnabled WebMail y en el menú lateral derecho entramos a enlaces de sitios. Luego entramos a "Agregar", y en la ventana emergente agregamos el nombre del host mail.alejo.dev.
